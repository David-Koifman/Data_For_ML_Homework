## 1. Описание задачи и датасета

**Задача ML:** Бинарная классификация тональности текстовых отзывов — `positive` / `negative`

**Модальность:** Текст

**Источники данных:**
| Источник | Тип | Записей |
|----------|-----|---------|
| HuggingFace `imdb` | Open dataset | 2000 |
| books.toscrape.com | Web scraping | 17 |
| **Итого после чистки** | | **1867** |

**Схема данных:**
| Колонка | Тип | Описание |
|---------|-----|----------|
| `text` | str | Текст отзыва |
| `label` | str | `positive` / `negative` |
| `source` | str | Источник данных |
| `collected_at` | str | Время сбора |
| `auto_label` | str | Метка агента |
| `confidence` | float | Уверенность агента (0–1) |

---

## 2. Что делал каждый агент

### DataCollectionAgent (Задание 1)
Собирает данные из 2+ источников и возвращает унифицированный DataFrame.
- **Решение:** IMDB как основной датасет (2000 отзывов), books.toscrape.com как второй источник (scraping)
- **Почему:** IMDB — стандартный бенчмарк для sentiment analysis, books.toscrape.com — публичный сайт без ограничений

### DataQualityAgent (Задание 2)
Выявляет и устраняет проблемы качества данных.
- **Найдено:** 150 выбросов по длине текста, дисбаланс 1.08x
- **Стратегия:** `clip_iqr` — удаление выбросов по IQR
- **Почему:** Очень короткие тексты не содержат контекста для sentiment, очень длинные — редкие случаи

### AnnotationAgent (Задание 3)
Автоматически размечает данные через Claude API (zero-shot).
- **Результат:** 1867 записей размечено, Cohen's κ = 0.809
- **Почему Claude API:** Не требует скачивания тяжёлых моделей, качество разметки выше чем TF-IDF
- **HITL:** 225 примеров с confidence < 0.7 флагируются для ручной проверки

### ActiveLearningAgent (Задание 4 — Трек A)
Умный отбор данных для обучения модели.
- **Модель:** Logistic Regression + TF-IDF
- **Стратегия:** Entropy — выбирает примеры где модель наименее уверена
- **Результат:** Entropy достигает accuracy=0.727, F1=0.747 на 150 примерах vs Random accuracy=0.673, F1=0.640

---

## 3. Human-in-the-Loop (HITL)

**Где происходит:** После Шага 3 (авторазметка)

**Что делает агент:**
- Флагирует примеры с `confidence < 0.7` → сохраняет в `review_queue.csv`
- Останавливает пайплайн и ждёт действия человека

**Что делает человек:**
1. Открывает `review_queue.csv`
2. Проверяет колонку `auto_label` для каждого примера
3. Исправляет ошибочные метки
4. Сохраняет файл как `review_queue_corrected.csv`
5. Нажимает Enter — пайплайн продолжается

**Статистика:**
- Флагов для проверки: 225 из 1867 (12%)
- После проверки: исправленные метки объединяются с уверенными

## 5. Ретроспектива

**Что сработало:**
- Claude API для авторазметки — высокое качество (κ=0.809) без обучения модели
- Entropy стратегия в Active Learning — стабильно превосходит Random
- Модульная архитектура — каждый агент независим и легко заменяем

**Что не сработало:**
- Scraping дал мало данных (17 записей) — books.toscrape.com не идеален для sentiment
- Дисбаланс данных небольшой, но может влиять на F1

**Что бы сделал иначе:**
- Взял бы второй источник с реальными отзывами (например, Yelp API или Twitter)
- Добавил бы больше итераций AL (10-20 вместо 5) для более чёткой кривой обучения
- Реализовал бы Streamlit дашборд для удобной HITL-разметки

## LLM в пайплайне (бонус +3)

Claude API используется в трёх агентах:
- **DataQualityAgent:** объясняет найденные проблемы и рекомендует стратегию чистки
- **AnnotationAgent:** авторазмечает тексты (zero-shot classification)
- **ALAgent:** анализирует результаты AL и даёт практические выводы
