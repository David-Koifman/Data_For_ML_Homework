# Skill: Pipeline (Мастер-скилл)

## Цель
Запустить полный дата-пайплайн используя 4 скилла в нужной последовательности с human-in-the-loop в ключевых точках.

## Запуск
Пользователь вызывает этот скилл и описывает задачу. Ты ведёшь его через весь пайплайн.

## Когда спрашивать пользователя (HITL-точки)
Спрашивай ТОЛЬКО на важных решениях:
1. **Старт** — какая ML-задача, какие данные нужны
2. **После сбора** — подтвердить найденные источники перед скачиванием
3. **После детекции проблем** — выбрать стратегию чистки
4. **После авторазметки** — проверить `review_queue.csv` и исправить метки
5. **Перед обучением** — подтвердить что данные готовы

НЕ спрашивай про: технические детали, форматы файлов, названия колонок

## Порядок действий

### Шаг 0: Понять задачу
Спроси пользователя:
```
Привет! Запускаю дата-пайплайн.
Опиши задачу: что хочешь классифицировать/предсказывать?
Например: "sentiment analysis отзывов на фильмы"
```

### Шаг 1: Сбор данных
Используй скилл `skills/data_collection/SKILL.md`:
- Найди подходящие источники для задачи пользователя
- Спроси подтверждение источников
- Собери данные
- Проведи EDA
- Сообщи результат и жди подтверждения перед следующим шагом

### Шаг 2: Чистка данных
Используй скилл `skills/data_quality/SKILL.md`:
- Обнаружь проблемы
- Покажи пользователю найденные проблемы с визуализацией
- Предложи стратегии чистки — **жди выбора пользователя**
- Примени выбранную стратегию
- Покажи таблицу до/после
- Сообщи результат

### Шаг 3: Авторазметка
Используй скилл `skills/annotation/SKILL.md`:
- Спроси сколько примеров разметить
- Запусти авторазметку
- Покажи метрики качества (Cohen's κ)
- **Остановись**: сохрани `review_queue.csv`
- Скажи пользователю: "Открой review_queue.csv, проверь метки с низкой уверенностью, сохрани как review_queue_corrected.csv"
- **Жди подтверждения** что проверка завершена

### Шаг 4: Active Learning
Используй скилл `skills/active_learning/SKILL.md`:
- Спроси стратегию
- Запусти AL-цикл
- Покажи кривые обучения
- Сохрани модель

### Шаг 5: Итоговый отчёт
Сгенерируй отчёт с метриками всех этапов:
```
Этап 1 (Сбор):     X записей из Y источников
Этап 2 (Чистка):   X записей осталось, удалено Y выбросов
Этап 3 (Разметка): Cohen's κ = X, флагов = Y
Этап 4 (AL):       accuracy = X, F1 = Y
```
Сохрани в `reports/` и обнови `README.md`.

- Если что-то пошло не так — сообщи пользователю и предложи повторить шаг
- Логируй метрики каждого этапа
