# Skill: DataQuality

## Цель
Выявить и устранить проблемы качества данных. Предложить пользователю стратегию чистки на основе найденных проблем.

## Когда спрашивать пользователя
- **Спроси стратегию чистки** — покажи найденные проблемы и предложи варианты, жди выбора
- **Спроси подтверждение** — перед применением чистки покажи что изменится
- НЕ спрашивай про технические детали реализации

## Порядок действий

1. **Загрузить данные** — из `data/raw/collected.parquet`

2. **Обнаружить проблемы** — запусти скрипт:
   ```
   skills/data_quality/scripts/data_quality_agent.py
   ```
   Метод `detect_issues(df)` вернёт отчёт о:
   - Пропущенных значениях
   - Дубликатах
   - Выбросах (IQR)
   - Дисбалансе классов

3. **Визуализировать проблемы** — построй 4 графика (по одному на каждую проблему)

4. **Спросить стратегию** — покажи пользователю найденные проблемы и предложи варианты:
   ```
   Найдено:
   - Выбросов: 150 (тексты аномальной длины)
   - Дисбаланс: 1.08x

   Варианты чистки:
   A) Агрессивная: удалить выбросы + дубликаты (потеряем 7% данных)
   B) Мягкая: только дубликаты, выбросы оставить

   Какую выбрать? (A/B)
   ```

5. **Применить чистку** — метод `fix(df, strategy)` с выбранной стратегией

6. **Сравнить результаты** — метод `compare(df_before, df_after)` → таблица до/после

7. **LLM объяснение** — вызови `llm_explain(report, task)` — Claude объяснит проблемы

8. **Сохранить** — чистый датасет в `data/raw/collected_clean.parquet`

9. **Сообщить итог** — сколько записей осталось, что было удалено и почему

## Выходной формат
```python
pd.DataFrame — чистый датасет
Сохранено в: data/raw/collected_clean.parquet
Отчёт: reports/quality_report.md
```

## Доступные скрипты
- `scripts/data_quality_agent.py` — агент (скиллы: detect_issues, fix, compare, llm_explain)
